# PAI: Plug-and-play Watermarking for AIGC Images
- **作者/年份/ venue**: （待补充） / （待补充） / （待补充）
- **核心问题**: 当前大多数AIGC服务基于扩散模型，需要一种无需额外编码器/解码器、即插即用的水印方案来实现版权保护和来源追溯。
- **方法分类**: 图像水印 / 扩散模型
- **核心创新**: 提出部署在提供商端的即插即用、无需训练的水印方案，与扩散生成框架完全兼容，支持版权追溯和篡改检测。
- **关键技术细节**:
  - 系统流程（三阶段）：
    1. **用户注册**：用户向提供商注册，提供商为其颁发唯一私钥K作为水印验证的唯一凭证。
    2. **带水印的生成**：用户使用私钥K和提示词访问提供商，生成带有水印的AIGC图像。
    3. **水印来源追溯**：在版权纠纷中，所有者提交带水印的图像和K进行来源追溯。服务提供商使用图像、元数据时间戳和K验证所有权，可追溯到原始用户。该方案还能检测图像是否遭受移除、伪造或篡改攻击，同时保持正确的所有权归属。
  - 特点：
    - 与扩散模型完全兼容，无需额外的编码器/解码器
    - 无需额外训练
    - 支持局部篡改检测：可提供二进制掩码定位篡改区域
- **实验设置**:
  - 数据集：AIGC图像
  - 评价指标：水印不可感知性、所有权验证准确率、篡改检测能力
  - 对比基线：（待补充）
  - 攻击类型：移除攻击、伪造攻击、篡改攻击（包括局部篡改）
- **优点/局限**:
  - 优点：
    - 即插即用，无需训练，部署成本低
    - 与现有扩散模型完全兼容
    - 支持所有权追溯和篡改定位
    - 可检测移除、伪造、篡改等多种攻击

- **我的思考/疑问**:
  - PAI的设计思路是从"提供商端"而非"用户端"出发，适合AIGC服务的商业模式。
  - 私钥K机制类似于数字签名的思路，确保了水印的唯一性和可追溯性。
  - 该方法是图像水印，但其即插即用的设计理念可能对音频水印领域也有借鉴意义。
  - 二进制掩码定位篡改区域的功能非常实用。
