# ZODiAC: Zero-shot Optimization-based Deep Image Anti-Copy
- **作者/年份/venue**: 2023 / NeurIPS 2023
- **核心问题**: 如何对已有图片（而非仅限扩散模型新生成的图片）嵌入鲁棒的扩散模型水印？Tree-Ring只能在生成阶段嵌入，无法处理已有的存量图片。
- **方法分类**: 图像水印 / 扩散模型 / 固有式水印
- **核心创新**: 基于Tree-Ring的理论基础，将水印嵌入扩展到**已有图片**的场景。不是在生成阶段嵌入，而是：先将已有图片通过DDIM反演转为噪声 → 在噪声空间嵌入环状水印 → 通过损失函数迭代优化 → 正向生成含水印图片。同时验证了扩散模型本身具有强大的攻击防御能力。
- **关键技术细节**:
  - **与Tree-Ring的关键区别**:
    - Tree-Ring：在扩散模型**生成图片阶段**嵌入水印（只能处理新生成的图片）
    - Zodiac：将**已有图片**输入扩散模型 → DDIM反演 → 嵌入水印 → 损失函数迭代（可处理任意存量图片）
  - **流程**：
    ```
    1. DDIM反演：将已有图片输入扩散模型，反演成高斯噪声状态
    2. 植入水印：在噪声空间的傅里叶域中嵌入环状水印（与Tree-Ring相同的方式）
    3. 正向生成：将含水印的噪声送入扩散模型正向生成图片
    4. 计算损失：用损失函数L计算新旧图片的差距
    5. 反向传播：将损失值反向传播回水印噪声图，不断迭代优化
    ```
  - **环状水印生成规则**：水印中每个元素的值取决于它在频域空间中距离中心点的距离。距离中心相同的所有点数值相同，形成一圈一圈的同心圆环。
  - **损失函数设计**：约束生成图片与原始图片的相似度，同时保证水印的可检测性
  - **扩散模型的攻击防御能力**：Zodiac的实验证明，通过扩散模型的生成过程嵌入的水印具有天然的鲁棒性，因为水印信息被"编织"进了图像的语义内容中
- **实验设置**:
  - 模型：Stable Diffusion
  - 基线：Tree-Ring
  - 攻击：各种图像变换和后处理攻击
- **优点/局限**:
  - 优点：
    - 可处理已有图片，不限于新生成的图片
    - 继承了Tree-Ring在傅里叶空间设计的鲁棒性
    - 迭代优化过程可以进一步提升水印质量
    - 验证了扩散模型作为水印载体的强大潜力
  - 局限：
    - 需要DDIM反演，增加了计算开销
    - 反演精度影响水印质量
    - 迭代优化过程较慢
- **我的思考/疑问**:
  - **PPT学习笔记**：PPT指出Zodiac来源于Tree-Ring但做了关键的工程整合——将水印嵌入从"生成时"扩展到"反演后"，这是一个重要的实用化改进
  - Zodiac证明了一个重要结论：扩散模型的去噪过程本身就具有"保护"水印的能力，因为水印信息被融入了图像的语义表示中
  - 与Gaussian Shading的对比：Gaussian Shading在分布层面保证无损，Zodiac在优化层面保证质量，两种思路各有侧重
  - 迭代优化的设计比Tree-Ring的一次性嵌入更灵活，但也更耗时
