# Watermark: Clean-label Backdoor Watermarking for Audio Dataset Protection
- **作者/年份/ venue**: （待补充） / （待补充） / （待补充）
- **核心问题**: 如何利用后门水印技术保护音频数据集的知识产权，同时避免传统后门水印的有害性、黑盒限制、音频质量损害和易被检测等问题。
- **方法分类**: 深度学习
- **核心创新**: 提出动态、无害、难泛化且保真的水印生成器，采用干净标签（Clean-label）策略，通过风格嵌入实现每个样本的水印动态变化。
- **关键技术细节**:
  - 后门攻击：将触发器放入数据集中，模型若用该数据集训练，模型会出现攻击者预设的目标。
  - 后门水印的缺陷：
    1. 有害后门：会产生可被攻击者利用的漏洞。
    2. 黑盒场景：需要访问模型内部参数。
    3. 损害音频质量：水印可能被人感知到。
    4. 容易被检测：固定模式下容易被发现。
  - 嵌入/提取流程（水印生成器训练）：
    1. 对于每个训练样本，随机选择一个参考音频。
    2. 使用GST-Tacotron2的风格编码器提取参考音频的风格嵌入。
    3. 将原始音频输入Wave-U-Net结构，并在下采样的每一层注入风格嵌入，使生成的水印音频在多个粒度上融入参考音频的风格特征。
    4. 通过Inversed Contrastive LTAF Loss让水印音频的说话人特征与原始音频显著不同，从而让未训练的模型难以正确识别。
    5. 通过Frequency Equalizer、Waveform Regulation、Semantic Regulation保持音频质量。
  - 数据集保护流程：
    - 随机选择样本（根据投毒率决定）。
    - 每个水印是动态的，取决于参考音频，即使同一说话人的不同样本水印也不同。
    - 不修改任何原始标签（干净标签）。
  - 验证流程：
    - 向可疑模型输入带水印的样本，同时输入该样本的干净版本。
    - 计算模型输出标签的概率。
    - 若对水印样本的真实标签概率高于对干净样本的真实标签概率或相当，则该模型使用了受保护的数据集。
- **实验设置**:
  - 数据集：语音数据集（Speaker Recognition任务）
  - 评价指标：保护有效性（验证准确率）、不可感知性（音频质量）
  - 对比基线：脏标签后门攻击（Dirty-label Backdoor Attack）
  - 攻击/验证类型：Speaker Recognition模型验证
- **优点/局限**:
  - 优点：
    - 采用干净标签策略，不修改原始标签
    - 水印动态生成（基于参考音频），不同样本水印不同，难以被检测和移除
    - 多层风格嵌入注入使水印难以被模型泛化
    - 通过多项正则化保持音频质量
    - 支持无害性验证（Harmless Verification）

- **我的思考/疑问**:
  - 该方法将后门攻击用于数据集保护，是一个有趣的应用方向。
  - 使用GST-Tacotron2和Wave-U-Net的组合架构，结合风格嵌入注入，实现了动态水印。
  - 干净标签策略避免了脏标签攻击中标签不一致的问题。
  - 需要关注水印的持久性和对不同说话人验证系统的泛化能力。
